# 运行问题故障排查指南

## 🔍 请先告诉我具体错误信息

运行不了可能有多种原因，请提供：
1. 使用的运行方式（HBuilderX 还是命令行）
2. 具体的错误提示信息
3. 是在哪个步骤失败的（编译、启动、打开浏览器等）

---

## 📋 常见问题及解决方案

### 问题 1：使用 HBuilderX 运行

#### 如果点击运行没有反应：
1. **检查项目类型**
   - 右键项目 → 重新识别项目类型 → 选择 "uni-app"
   
2. **检查是否安装了插件**
   - 工具 → 插件安装
   - 确保已安装 "uni-app" 相关插件

3. **清理缓存**
   - 项目 → 清理项目缓存
   - 重新运行

#### 如果提示找不到文件：
- 确保项目根目录有 `manifest.json`、`pages.json`、`main.js`、`App.vue` 等文件

---

### 问题 2：使用命令行运行

#### 如果提示 "uni: command not found"：

**解决方案：**

1. **安装 uni-cli（推荐）**
   ```bash
   npm install -g @dcloudio/uni-cli
   ```

2. **或者使用 npx**
   ```bash
   npx @dcloudio/uni-cli dev:h5
   ```

3. **或者修改 package.json 使用 vite**
   
   将 scripts 修改为：
   ```json
   {
     "scripts": {
       "dev:h5": "vite",
       "dev:mp-weixin": "vite build --watch --mode mp-weixin",
       "build:h5": "vite build"
     }
   }
   ```

#### 如果提示依赖未安装：
```bash
npm install
```

#### 如果端口被占用：
修改 `vite.config.js` 中的端口：
```javascript
server: {
  port: 8081,  // 改为其他端口
}
```

---

### 问题 3：编译错误

#### 如果提示 "缺少 index.html"：
- ✅ 已修复：项目中已有 `index.html` 文件

#### 如果提示语法错误：
1. 检查控制台具体错误信息
2. 检查 Vue 文件语法是否正确
3. 检查 JSON 配置文件格式是否正确

---

### 问题 4：页面空白

#### 可能原因：
1. **路由配置问题**
   - 检查 `pages.json` 中页面路径是否正确
   
2. **首页路径错误**
   - `pages.json` 中第一个页面路径必须正确存在

3. **CSS 样式问题**
   - 检查是否有样式冲突
   - 检查是否使用了不支持的 CSS 属性

---

## 🚀 推荐运行方式

### 方式 1：HBuilderX（最简单）

1. 下载安装 HBuilderX
2. 导入项目
3. 右键项目 → 运行 → 运行到浏览器

### 方式 2：命令行（需要 Node.js）

1. 安装依赖：
   ```bash
   npm install
   ```

2. 如果 `uni` 命令不存在，安装：
   ```bash
   npm install -g @dcloudio/uni-cli
   ```

3. 运行：
   ```bash
   npm run dev:h5
   ```
